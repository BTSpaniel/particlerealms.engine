Documentation Modules

Engine v2 is organized into three main phases, each with its own documentation module. Start with Core for GPU fundamentals, then explore ECS for entity systems, and finally Render for the graphics pipeline.

Phase 1: Core (GPU & Runtime)

Foundation layer covering GPU device management, buffer allocation, shader compilation, and the main event loop. Essential for understanding how the engine communicates with WebGPU.

‚Üí View Core Documentation

Phase 2: ECS v2 (Entity Component System)

High-performance entity component system with worlds, entities, components, queries, and systems. Provides the architecture for game logic and simulation.

‚Üí View ECS Documentation

Phase 3: Render (Graphics Pipeline)

3D rendering pipeline with materials, meshes, cameras, and render passes. Integrates with ECS for entity-driven rendering.
